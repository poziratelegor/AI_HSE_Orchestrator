# ADR 003 — RAG approach

## Decision
Используем Supabase Storage + Postgres + pgvector для документов и retrieval.

## Why
- один стек для auth, DB и vector storage
- удобно для MVP
- меньше интеграций

## Consequences
- нужно аккуратно настроить ingestion pipeline
- качество retrieval зависит от chunking strategy
