# ADR 002 — Orchestrator approach

## Decision
Стартуем с гибридного rule-based оркестратора, затем добавляем LLM classifier.

## Why
- быстрее собрать MVP
- легче дебажить
- проще объяснять на защите

## Consequences
- на раннем этапе будет ограниченная точность
- потребуется постепенное усложнение логики
