# ADR-001 — Выбор технологического стека

**Статус:** принято  
**Дата:** 2025-04

---

## Контекст

Необходимо выбрать стек для AI-оркестратора студенческих задач. Ключевые требования:

- Быстрый выход на MVP (учебный проект с жёсткими сроками)
- Единый репозиторий — веб-фронтенд, API и фоновые задачи в одном месте
- Встроенная аутентификация и хранение данных без отдельного бэкенд-сервиса
- Поддержка векторного поиска для RAG-пайплайна
- Возможность деплоя на Vercel без DevOps-усилий
- Второй канал взаимодействия — Telegram Bot

---

## Решение

**Принятый стек:**

| Слой | Технология | Обоснование |
|---|---|---|
| Фреймворк | Next.js 15 (App Router) | SSR + route handlers + Turbopack в одном пакете |
| Язык | TypeScript 5.7 (strict) | Типобезопасность от API-схем до UI-компонентов |
| База данных | Supabase (PostgreSQL 15) | Auth + DB + Storage + RLS в одном сервисе |
| Векторный поиск | pgvector | Нативная интеграция с Postgres, не нужен отдельный векторный DB |
| LLM | OpenAI GPT-4o-mini | Лучшее соотношение цена/качество для классификации и генерации |
| Эмбеддинги | text-embedding-3-small | 1536 измерений, стоимость $0.02/M токенов |
| STT | OpenAI Whisper | Единый провайдер с LLM, хорошее качество для русского языка |
| Стили | Tailwind CSS 3 + CSS-переменные | Утилитарный подход, дизайн-система через переменные |
| Кэш | Upstash Redis (REST) | Serverless Redis без постоянного TCP-соединения, работает в Edge |
| Мониторинг | Sentry | Стандарт де-факто для Next.js |
| Бот | Telegram Bot API | 700 млн+ активных пользователей, простой webhook-интерфейс |
| Деплой | Vercel + Docker | Vercel для прода, Docker для локальной разработки |

---

## Рассмотренные альтернативы

### Отдельный Express/FastAPI бэкенд
**Отклонено.** Дополнительный сервис = дополнительный деплой, дополнительная сеть, дополнительная точка отказа. Next.js route handlers покрывают все потребности MVP без оверхеда.

### Firebase вместо Supabase
**Отклонено.** Отсутствует поддержка pgvector. Firestore — документная БД, что усложнило бы реляционные запросы. Supabase даёт полноценный PostgreSQL.

### Pinecone / Weaviate для векторного поиска
**Отклонено.** Отдельный сервис увеличивает стоимость и сложность. pgvector в составе Supabase достаточен для объёмов MVP (< 1M чанков) и не требует дополнительной инфраструктуры.

### Anthropic Claude вместо OpenAI GPT-4o-mini
**Отложено.** Архитектура поддерживает смену провайдера через `LLM_PROVIDER` env. OpenAI выбран как первый провайдер из-за Whisper STT — нет смысла иметь два LLM-провайдера одновременно на MVP.

### @upstash/redis SDK вместо REST API
**Отклонено.** SDK требует постоянного соединения, несовместимого с serverless-средой Vercel Edge. Upstash REST API работает через обычный `fetch`, что проще и надёжнее.

## Последствия

### Положительные
- Единый монорепозиторий без микросервисной сложности
- Supabase берёт на себя auth, RLS, Storage и pgvector
- Next.js route handlers выполняются в serverless-среде — масштабирование бесплатно
- Typesafety от схемы БД (через generate_typescript_types) до UI

### Отрицательные и риски
- Бизнес-логика в route handlers требует дисциплины для предотвращения «fat handler» антипаттерна → решено через слой `lib/services/*`
- Vercel Hobby ограничивает время выполнения функций 10 секундами → критично для Whisper → нужен Pro-план или вынос тяжёлых операций в background jobs
- pgvector при объёмах > 10M векторов потребует миграции на специализированное решение
